Calling a function via the dot operator







An object observation:
Take a look at Code Block BBBb. By inspection, the entire subPrimeFunc is configured like an object with name:value pairs. Look at the contents of the return statement and notice the open and close braces, and within the braces are the names: response1, response2, and response3. And the corresponding values are the functions and response3's value is the invoked function "response3Function()". Look how these name:values are called: via the dot operator. It all looks like an object to me. Yes I know functions are considered objects.
But this is a great example of a function having a familiar object look.
